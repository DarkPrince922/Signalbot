# Signalbot — Telegram-бот сигналов по крипто-фьючерсам (signal-only)

Личный инструмент для **исследования и сравнения** торговых стратегий на
крипто-фьючерсах. Бот **не торгует** и **не отправляет ордера на биржу** —
он считает сигналы, шлёт уведомления в Telegram, ведёт честный бэктест и
форвард-трекер живых сигналов.

> Это инструмент для анализа, а не финансовый совет. Решения о реальных
> сделках и риске — на стороне пользователя.

## Что внутри

- **Только закрытые свечи** — никакого look-ahead bias (есть тесты).
- **Честный бэктест** — комиссии (maker/taker), проскальзывание, опционально
  фандинг; обязательный out-of-sample сплит.
- **Биржа-агностик** — источник данных за интерфейсом `DataProvider`, дефолт
  Binance USDⓈ-M через `ccxt`, меняется в конфиге.
- **Одно-пользовательский режим** — бот реагирует только на whitelisted
  `TELEGRAM_USER_ID`.
- **Стратегии-плагины** — новая стратегия = один файл в `signalbot/strategies/`,
  без правки ядра (авто-регистрация).

## Архитектура

```
signalbot/
├── core/      config, models, db, types
├── data/      DataProvider + ccxt-реализация с кэшем
├── strategies/ base + 6 стратегий + registry + indicators
├── engine/    backtester, metrics, live_engine, tracker, plotting
└── bot/       aiogram main, handlers, formatters
```

Индикаторы реализованы нативно на pandas/numpy (`strategies/indicators.py`),
чтобы не тянуть бинарную зависимость pandas-ta и оставаться совместимыми с
современными версиями numpy. Все они причинные (causal) — значение на свече `i`
не зависит от будущих свечей.

### Стартовые стратегии

Все они — **болванки для проверки механики, не торговые рекомендации**.
Параметры тюнятся в `config.yaml`, каждая включается/выключается там же или
из бота. По умолчанию включены только `ema_cross` и `rsi_reversion`.

| Стратегия | Идея |
|---|---|
| `ema_cross` | пересечение быстрой/медленной EMA, стоп по ATR, тейк по R |
| `rsi_reversion` | возврат RSI из зоны перекупленности/перепроданности + фильтр тренда |
| `donchian_breakout` | пробой N-периодного канала Дончиана, трейл по ATR |
| `macd_cross` | пересечение линии MACD и сигнальной, стоп по ATR |
| `bollinger_reversion` | возврат цены внутрь полос Боллинджера (mean-reversion), тейк к средней |
| `supertrend` | вход на смене направления Supertrend, стоп по линии индикатора |

## Установка и запуск

```bash
cp .env.example .env          # вписать TELEGRAM_BOT_TOKEN и TELEGRAM_USER_ID
pip install -r requirements.txt
pip install -e .
python -m signalbot.bot.main
```

Или через Docker:

```bash
cp .env.example .env
docker compose up --build -d
```

## Конфиг

Секреты — в `.env` (`TELEGRAM_BOT_TOKEN`, `TELEGRAM_USER_ID`, `EXCHANGE_ID`).
Поведение — в `config.yaml` (пары, стратегии и их параметры, косты, сплит
out-of-sample, таймаут трекера). Включённые стратегии / пары / флаг
уведомлений можно менять прямо из бота — они хранятся в таблице `settings`
и переживают перезапуск.

## Если «нет данных для анализа»

Это значит, что не удалось получить котировки с биржи. Бот теперь показывает
конкретную причину прямо в ответе (и пишет её в лог). Частые случаи:

- **Биржа недоступна с сервера.** Binance блокирует часть облачных/региональных
  IP (ошибки вида `NetworkError ... fapi.binance.com`, 451). Решение — VPN/прокси
  или другая биржа: смени `EXCHANGE_ID` в `.env` (например, `bybit`, `okx`,
  `gateio`).
- **Формат пары.** Для USDⓈ-M фьючерсов символ автоматически приводится к виду
  `BASE/QUOTE:QUOTE` (`BTC/USDT` → `BTC/USDT:USDT`), но монета должна реально
  торговаться на выбранной бирже.

## Анализ «куда ставить сейчас» + определение риска

Кнопка **🔎 Анализ сейчас** (или `/scan`) сканирует все отслеживаемые пары в
текущий момент и для каждой даёт читаемый перевес, даже если ни одна дискретная
стратегия не сработала. Направление определяется голосованием четырёх
индикаторов — тренд EMA, MACD, RSI, Supertrend; доля согласия даёт
«уверенность» (score).

Для выбранной пары бот **сам определяет риск сделки** и подгоняет размер:

- **волатильность** — ATR как % от цены (низкая/средняя/высокая);
- **согласованность** сигналов (слабый перевес = выше риск);
- **живой винрейт** стека стратегий из форвард-трекера (если набралось ≥10 сделок).

Из этого складывается уровень риска 🟩 LOW / 🟨 MEDIUM / 🟥 HIGH, и
рекомендованный риск на сделку масштабируется множителями из
`config.yaml` → `analysis` (`size_factor_*`). Так размер позиции в USDT/контрактах
считается уже с учётом риска. Кнопкой **✅ Отслеживать** выбранную идею можно
записать как виртуальный сигнал — дальше её судьбу ведёт форвард-трекер теми же
метриками.

Всё это — подсказка для решения, **не финансовый совет**; исполнение и риск
остаются за пользователем.

## Динамика рынка в целом

Кнопка **🌐 Рынок в целом** (или `/market`) оценивает не отдельную пару, а
крипторынок целиком по более широкому набору монет (`config.yaml` →
`analysis.universe`, по умолчанию топ-8; если пуст — берутся отслеживаемые пары):

- **режим рынка** — 🟢 RISK-ON / 🔴 RISK-OFF / 🟡 MIXED, по ширине рынка
  (сколько монет в аптренде) и среднему движению;
- **волатильность** — спокойная / обычная / турбулентная по среднему ATR%;
- **ширина рынка** — % монет в аптренде (🟢/🔴/⚪ разбивка);
- **среднее движение** за окно (`analysis.lookback_bars`) и **BTC как ориентир**;
- **лидеры и аутсайдеры** — топ растущих и падающих монет.

Для каждой монеты также считается **динамика движения**: изменение за окно и
ярлык моментума (🚀 сильный рост / 📈 рост / ➡️ боковик / 📉 снижение /
🔻 сильное падение) — он же показывается в карточке пары в `/scan`.

## Бэктест и оценка

Бэктестер идёт по свечам от `warmup` до конца, на каждом шаге вызывает
`strategy.generate(df.iloc[:i+1])` (только прошлое). При сигнале моделирует
сделку: вход по close текущей свечи или open следующей (`fill_mode`), затем по
будущим свечам ищет, что наступит раньше — SL или TP. **Если оба уровня задеты
внутри одной свечи — консервативно считаем LOSS.** Учитываются комиссии,
проскальзывание и (опционально) фандинг. Метрики выводятся отдельно по
in-sample и out-of-sample.

**Форвард-трекер важнее бэктеста.** Каждый живой сигнал сохраняется как
виртуальная сделка; на новых свечах трекер фиксирует её судьбу (WIN/LOSS/EXPIRED)
и копит живую статистику теми же метриками. Доверять стратегии стоит только
после десятков живых сделок, а не после красивой кривой на истории.

## Тесты

```bash
pytest
```

Покрыто, как требует ТЗ: отсутствие look-ahead (индикаторы причинны +
бэктестер видит только префиксы + будущая свеча не меняет прошлый сигнал),
расчёт SL/TP-исхода включая «обе цели в одной свече → LOSS», метрики на
синтетике с известным ответом, дедуп сигналов на одной свече, whitelist.

## Вне scope

Автоторговля, отправка ордеров, хранение торговых ключей, монетизация/рассылка
сигналов другим людям — **не делаются намеренно**.
